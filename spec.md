# BANA 7025 Python Course Platform Architecture Spec

## 1. Objective

Revamp BANA 7025 from an R-focused graduate data wrangling / introductory data science course into a Python-first course delivered through a modern course platform.

The platform should support:

1. A Quarto-based online textbook
2. Hosted Quarto Reveal.js slides
3. Python-based example notebooks
4. Lab notebooks
5. Tuesday-style interactive coding notebooks
6. A semester-long project structure
7. An AI tutor grounded in the course content
8. A deployment path that can eventually run through Vercel or similar hosting

The goal is not just to convert R content to Python. The goal is to create a more coherent student experience where students read, watch, practice, ask questions, and build toward a final project in one integrated ecosystem.

---

## 2. Design Principles

### 2.1 GitHub is the Source of Truth

All course content should live in the repository:

* Quarto textbook chapters
* Quarto slide decks
* Jupyter notebooks
* labs
* assignments
* project materials
* datasets
* AI tutor configuration
* deployment scripts

The deployed website should be generated from the repository.

### 2.2 Quarto + Python is the Authoring Layer

Course chapters and slides should be written in Quarto using executable Python code blocks.

The authoring workflow should support:

* `.qmd` textbook chapters
* `.qmd` Reveal.js slides
* executable Python examples
* rendered HTML output
* reproducible builds
* clear separation between student-facing and instructor-only content

### 2.3 Vercel or Similar Platform is the Deployment Layer

The static course site can be generated by Quarto and deployed to Vercel, Netlify, GitHub Pages, or another static hosting option.

The AI tutor requires server-side compute. If using Vercel, the tutor backend should run as an API route or serverless function.

### 2.4 AI Tutor Should Be Grounded and Guardrailed

The AI tutor should use only approved student-facing course content.

It should help students understand concepts, debug code, and reason through problems, but it should not provide direct answers to graded assignments, quizzes, exams, or final project deliverables.

## 5. Course Content Model

Each major topic should have five connected artifacts:

1. Textbook explanation
2. Tuesday slide deck
3. Tuesday coding notebook
4. Thursday lab notebook
5. Project checkpoint

Example:

```text
Topic: pandas groupby

Textbook:
- Explain split-apply-combine
- Show examples

Slides:
- Introduce groupby conceptually
- Walk through simple code

Tuesday notebook:
- Guided examples
- Short Your Turn exercises

Lab:
- Applied business scenario

Project:
- Aggregate customer behavior metrics
```

This creates repetition without feeling redundant.

## 6. Semester-Long Project

BANA 7025 should include a semester-long project that builds incrementally.

The project should be introduced early, ideally Week 2.

Potential project theme:

```text
Customer analytics / retail transaction analysis / subscription churn / marketing campaign performance
```

The project should support:

* importing data
* inspecting data
* cleaning columns
* handling missing values
* filtering and sorting
* aggregating metrics
* joining multiple tables
* reshaping data
* visualizing patterns
* writing business conclusions
* optional introductory modeling

Example project progression:

```text
Week 1: Course setup and Python environment
Week 2: Load project dataset
Week 3: Inspect rows, columns, data types
Week 4: Filter and subset observations
Week 5: Create summary metrics
Week 6: Clean missing and messy values
Week 7: Join multiple tables
Week 8: Reshape data for analysis
Week 9: Visualize business patterns
Week 10: Create reusable functions
Week 11: Prepare final analytic dataset
Week 12: Optional baseline model or scoring logic
Week 13: Final project refinement
Week 14: Final project submission
```

The final project should feel like assembling and improving work students have already practiced.

## 7. AI Tutor Architecture

The AI tutor should be embedded in the course platform.

### 7.1 Tutor Flow

```text
Student asks question
в†“
Frontend sends question to /api/chat
в†“
Backend retrieves relevant course content
в†“
Backend builds grounded prompt
в†“
Backend calls LLM API
в†“
Tutor returns response with references to course materials
```

### 7.2 Tutor Should Use

```text
Allowed Sources
- book/**/*.qmd
- slides/**/*.qmd
- notebooks/examples/**/*.ipynb
- notebooks/tuesday-your-turn/**/*.ipynb
- labs/student-facing notebooks
- project/student-facing materials
- public assignment instructions
- dataset documentation
```

### 7.3 Tutor Should Exclude

```text
Restricted Sources
- answer keys
- solution notebooks
- quiz answers
- exams
- grading rubrics not shared with students
- instructor planning docs
- private notes
```

### 7.4 Tutor Behavior

The tutor should:

* explain concepts clearly
* use beginner-friendly Python examples
* reference course materials when possible
* help debug code
* ask guiding questions
* provide hints before full solutions
* encourage students to reason through assignments

The tutor should not:

* complete graded homework
* provide quiz answers
* write final project submissions
* expose answer keys
* reveal instructor-only materials

  "module": "Data Wrangling",
  "week": 5,
  "topic": "groupby",
  "source_type": "textbook",
  "path": "book/07-wrangling-data.qmd",

## 11. Initial BANA 7025 Python Course Direction

The course should be designed as a graduate-level introduction to Python for data science and data wrangling.

Recommended course emphasis:

```text
1. Python environment and notebooks
2. Python fundamentals
3. pandas DataFrames
4. importing data
5. cleaning messy data
6. filtering, sorting, selecting
7. creating and transforming columns
8. grouping and aggregation
9. joining data
10. reshaping data
11. working with dates and strings
12. visualization
13. exploratory data analysis
14. functions and reproducible workflows
15. optional intro to modeling or analytics pipeline
```

The course should avoid becoming too algorithm-heavy. BANA 7025 should prepare graduate students to work with data in Python and be ready for later courses involving machine learning, AI systems, or advanced analytics.
